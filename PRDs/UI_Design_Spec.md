# 法律 AI 助手：UI 设计规范与端侧交互指南

本文档与《[Product_System_Architecture.md](./Product_System_Architecture.md)》**分期与范围口径一致**：第一期优先 **金额计算** 与 **提醒**；**知识库 UI 不出现**（第二期）；**搜索类 UI 待定**；首发 **单律师测试**。金额计算字段与业务规则以《[Legal_Logic_Implementation.md](./Legal_Logic_Implementation.md)》为准；涉及 **workbench 已实现页面** 时与架构文档 **§2.4** 同步更新。

---

## 1. 设计语言

参考业内「专业工具型」极简风格（如 Alpha 类产品思路）：**严谨、干净、高效**，信息密度可控。

### 1.1 色彩（Design Tokens）

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-primary` | `#1D4ED8` | 主按钮、主链接、导航选中、关键操作 |
| `--color-bg-page` | `#F8FAFC` | 页面底色 |
| `--color-bg-surface` | `#FFFFFF` | 卡片、表格容器背景 |
| `--color-border-subtle` | `#E2E8F0` | 分割线、表格线（1px） |
| `--color-text-primary` | `#0F172A` | 主文案 |
| `--color-text-secondary` | `#64748B` | 辅助说明、表头次要信息 |
| `--color-warning` | `#F59E0B` | 续封提醒、即将到期、需人工复核 |
| `--color-danger` | `#DC2626` | 已超期、严重错误（慎用） |
| `--color-success` | `#16A34A` | 已完成、已确认（可选） |

### 1.2 字体与排版

- **字体**：系统 UI 字体栈（Windows：微软雅黑；macOS：PingFang SC）。
- **字号**：正文 `14px`，说明 `12px`，标题 `16px`–`18px`。
- **行高**：正文 `line-height: 1.6`。
- **数字**：金额、案号、日期建议 `tabular-nums`，便于纵向对齐。

### 1.3 圆角与阴影

- 卡片圆角 `8px`；标签小圆角 `4px` 或 pill。
- 阴影仅用于下拉层或悬浮卡片，宜轻。

## 2. Web 端布局（桌面）

### 2.1 整体结构

- **左侧**：固定窄栏图标导航（可折叠），约 `56px`–`72px`。
  - **第一期**：不出现「知识库」「所内检索」「法规/智能问答」入口；菜单项与**当期实际上线模块**一致（如计算、提醒、待办等），**勿挂死链**。
  - **第二期**：在同一侧栏结构上增加知识库项即可。
- **顶栏**：案号或面包屑、用户菜单。
  - **全局搜索（顶栏搜索按钮/框）**：**待定**。第一期以计算与提醒为主时可**不呈现**；与案件列表一并规划后再定（与架构文档「搜索能力待定」对齐）。
- **主内容区**：白底卡片 + 页面灰底层次。

#### 2.1.1 金额计算原型（workbench，与架构 §2.4 一致）

全量愿景为 **侧栏 + 顶栏 + 主内容区**（见上）。**当前仓库原型**为单栏 **单页 + Tabs**，不实现侧栏，避免与「未交付模块」产生死链；后续接入 `ProLayout` 时可将 Tabs 迁入「新建/编辑计算」路由或侧栏「工具」分组。

| Tab 标签 | 用户目标 | 对接 API（`Content-Type: application/json`） | 客户端下载文件名（示意） |
|----------|----------|-----------------------------------------------|---------------------------|
| 民间借贷 | 利息分段试算与导出 | `POST /api/calculate`、`POST /api/export/excel` | `民间借贷计算书.xlsx` |
| 房屋租赁 | 滞纳金 + 占用费试算与导出 | `POST /api/rental/calculate`、`POST /api/rental/export/excel` | `房屋租赁计算书.xlsx` |

- **表单校验**：租赁场景下，若未填「实际搬离日」，则须填「起诉日」（与 `RentalRequest` 及 Legal 文档 §0 备忘 8.x 一致）；前端宜在提交前提示，与后端校验口径一致。
- **结果区**：展示与后端 `CalculationResult` 一致的明细表（费用类目、基数、利率标准、起止日、天数、金额）及 `rule_version`、`assumptions_used`；租赁侧若有 `messages` 一并展示。
- **源码位置**（便于走查）：`web/src/App.tsx`（标题 + Tabs）、`web/src/PrivateLendingPanel.tsx`、`web/src/RentalPanel.tsx`、`web/src/calcShared.tsx`。

### 2.2 列表页

- **布局**：卡片列表或「卡片 + 表格」；可参考 `image_0ae0e1.png`（若仓库暂无，定稿后放入 `PRDs/assets/` 并更新路径）。
- **表格**：Ant Design `Table`，支持列排序、分页或虚拟滚动。
- **列表内搜索框、表头筛选**：**待定**——与顶栏搜索一致；第一期不做完整案件列表时可省略。
- **状态标签**：「进行中」「已结案」「待续封」等，浅色底 + 深色字，圆角 `4px`。

### 2.3 证据处理区（双栏，按迭代上线）

- **左侧**：文件预览（PDF/图片/音频）。
- **右侧**：AI 抽取字段表单，机器字段带「需确认」提示，可恢复模型原文。
- **底部**：保存草稿、标记已核对、导出（导出格式可二期加强）。

### 2.4 空态、加载与错误

- **空列表**：说明 + 主操作按钮（与当期主链路一致，如「新建提醒」「新建计算」）。
- **加载**：表格骨架屏优于整页 Spin；长任务提示可离开页面。
- **错误**：`Alert` / `Message`；字段级 `help` 简短可执行。

### 2.5 第一期侧重与「搜索」结论

- **与架构文档对齐**：第一期优先 **金额计算** 与 **提醒**；完整案件列表、全局搜索、复杂筛选 **不强制同迭代交付**。
- **搜索**：未定迭代前 **不要**放置不可用的搜索占位。
- **知识库**：第一期 **无任何入口**（见 §6.1）。

## 3. 小程序端布局

### 3.1 首页

- **金刚区**：如「快速计算」「待办提醒」等，**按第一期实际上线项裁剪**；可含「拍照识别」若文书 OCR 当期要做。
- **信息优先级**：待办角标 > 最近动态 > 营销位（第一期可无营销位）。
- **搜索**：与 Web 相同，**待定**，避免空功能。

### 3.2 提醒流（时间轴）

- 左侧竖线 + 右侧卡片；临近到期用警示橙；超时强化警示。
- 卡片可进详情；支持「标记已处理」写回服务端。

### 3.3 表单

- 多用 Picker / Calendar，减少手输；金额与日期修改前二次确认。

## 4. 组件选型

| 端 | 库 | 说明 |
|----|-----|------|
| Web | Ant Design 5.x + React 18 | `ProLayout`、`Table`、`Form`、`Upload`、`DatePicker` |
| Web 样式 | 可与 Tailwind 并存 | 主题以 Ant Design Token 为准，避免双色冲突 |
| 小程序 | Vant Weapp | `Cell`、`Field`、`Calendar`、`Notify`、`Uploader` |

锁定 minor 版本，大版本升级前回归。

## 5. 可访问性与国际化（建议）

- 对比度建议满足 WCAG AA。
- 键盘焦点可见；错误文案说明「用户可执行的下一步」。
- 中文首发；关键文案抽常量便于扩展。

## 6. Web 信息架构（与分期对齐）

以下路由为**全量愿景**；第一期仅实现其中与 **计算 + 提醒** 相关的子集即可，其余可后置。

| 路由级页面 | 说明 |
|------------|------|
| 登录 / 选择组织 | 单律师测试时可极简 |
| 案件列表 | 新建、状态标签、表格列；**搜索与筛选待定** |
| 案件详情（Tab） | 概览、计算记录、文书与时限、证据与时间轴（按迭代开放 Tab） |
| 新建/编辑计算 | 表单 + 结果预览区；业务未定稿前可用 Mock。**当前原型**：见 §2.1.1，以 **民间借贷 / 房屋租赁** 双 Tab 直连真实 API，非 Mock |
| 文书上传与 OCR 校对 | 按迭代 |
| 设置 | 通知开关等 |

### 6.1 知识库 / 检索问答（第二期）

- **第一期**：侧栏、顶栏、案件详情、小程序金刚区 **均不出现** 知识库/问答/法规检索类入口。
- **保留**：菜单配置中可用 `hidden: true` 预留 `key`，第二期打开即可。
- **第二期**：再补列表、上传、检索结果、引用面板等稿与规范。

## 7. Cursor 设计指令（示例）

> 请使用 React 与 Ant Design，按本规范色板与布局实现 Web 端页面：顶栏面包屑、左侧窄导航、主区含与「金额计算」或「提醒待办」相关的表单与列表。**搜索框与表头筛选仅在产品明确要求本期交付时再加入**；否则不要放置搜索占位。背景 `#F8FAFC`，主内容白底卡片，边框 `#E2E8F0`。
>
> **若本期仅为计算器原型（workbench）**：可采用单页 **Tabs** 切换「民间借贷」「房屋租赁」，主色 `#1D4ED8`，开发代理 `/api` 至后端；不要求左侧导航，见 §2.1.1。

（若本期做案件列表且确认需要搜索，再在指令中显式加上搜索框与筛选条件。）
